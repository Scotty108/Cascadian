# Cascadian Documentation

This directory contains all project documentation organized by category.

## Directory Structure

### `/architecture/`
Core architectural specifications and system designs
- Database schemas and structure
- Strategy builder specifications
- System architecture plans
- Technical specifications

### `/guides/`
Implementation guides, tutorials, and how-tos
- Feature implementation guides
- Quick start guides
- Migration guides
- Deployment guides
- System formatting guides

### `/runbooks/`
Operational runbooks and procedures
- Smart Money Flow runbook
- System maintenance procedures

### `/database/`
Database-specific documentation
- Database structure documentation
- Query references
- Investigation findings
- Data issues and resolutions

### `/migrations/`
Migration-related documentation
- Migration instructions
- Migration history and notes

### `/api/`
API documentation and specifications
- Endpoint documentation
- API integration guides

### `/archive/`
Historical documentation and completed work

#### `/archive/completed-features/`
Documentation for completed features and implementations

#### `/archive/session-reports/`
Historical development session reports

#### `/archive/historical-status/`
Point-in-time status reports and progress updates

## Existing Documentation

See also these other documentation directories:
- `/docs/ui-*` - UI/UX documentation
- `/docs/wallet-*` - Wallet system documentation
- `/docs/POLYMARKET_*` - Polymarket integration docs
- `/docs/BULK_SYNC_*` - Bulk sync system docs

## Contributing

When adding new documentation:
1. Use clear, descriptive filenames
2. Place files in the appropriate subdirectory
3. Update this index if adding new categories
4. Move completed project docs to `/archive/completed-features/`
5. Follow existing naming conventions:
   - Guides: `*-guide.md`
   - Runbooks: `*-runbook.md`
   - Architecture: `*-architecture.md` or `*-spec.md`
